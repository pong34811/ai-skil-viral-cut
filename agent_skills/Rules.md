# Rules & Constraints: Viral Cut Detection

> ไฟล์นี้เป็นส่วนหนึ่งของ [[Skills]] — ดู Workflow ที่ [[WorkFlows]]

---

## 1. ข้อจำกัดด้านเนื้อหา (Content Constraints)

### 1.1 เนื้อหาที่ห้ามเลือกเด็ดขาด
- ละเมิดลิขสิทธิ์เพลง, คลิป, หรือสื่อของผู้อื่น
- เนื้อหาที่ผิดกฎหมายทุกประเภท
- ภาพหรือเสียงที่ลามกอนาจาร
- การยุยงให้เกิดความรุนแรงหรือเกลียดชัง

### 1.2 เนื้อหาที่ควรหลีกเลี่ยง
- ความรุนแรงที่มากเกินจำเป็น (เลือด, การทำร้ายร่างกาย)
- การละเมิดสิทธิส่วนบุคคล (เบอร์โทร, ที่อยู่, ข้อมูลส่วนตัว)
- ประเด็นละเอียดอ่อนทางสังคม: การเมือง, ศาสนา, เชื้อชาติ
- เนื้อหาที่อาจสร้างความเข้าใจผิดหรือเป็นเท็จ (misinformation)
- การใช้คำหยาบหรือภาษาที่ไม่เหมาะสมในปริมาณมาก

### 1.3 ข้อยกเว้น
- หากเนื้อหาที่ sensitive มีคุณค่าทางสาระหรือการศึกษา และไม่ได้สร้างความเสียหาย สามารถเลือกได้โดยต้องระบุคำเตือน

---

## 2. ข้อจำกัดด้านระยะเวลา (Duration Rules)

### 2.1 ระยะห้ามต่ำกว่า
- **Viral Cut ทุกประเภทต้องมีความยาว ≥ 40 วินาที เสมอ**
- ถ้าช่วงที่น่าสนใจสั้นกว่า ให้ขยายขอบเขตให้ครบ 40 วินาที โดยรักษาบริบทโดยรอบ

### 2.2 Short-form Content
| แพลตฟอร์ม | ระยะเวลาแนะนำ |
|-----------|-------------|
| TikTok | 40 วินาที - 3 นาที |
| YouTube Shorts | 40 วินาที - 3 นาที |
| Instagram Reels | 40 วินาที - 3 นาที |

### 2.3 Long-form Content
- ระยะเวลา: 3-10 นาที
- เหมาะสำหรับ YouTube, Facebook

---

## 3. ข้อจำกัดด้าน Timeline (Timeline Rules)

- **ห้าม** เลือก cut ที่มี timeline ซ้อนทับกัน
- ถ้า cut มีระยะห่างน้อยกว่า 30 วินาที ให้พิจารณารวมเป็น cut เดียว
- แต่ละ cut ควรกระจายตัวทั่วทั้งวิดีโอ ไม่กระจุกอยู่เฉพาะช่วงใดช่วงหนึ่ง

---

## 4. หลักเกณฑ์การตัดสินใจ (Decision Rules)

### 4.1 Tie-breaker (เมื่อคะแนนเท่ากัน)
1. เลือก cut ที่เกิดจาก **Chaos / Conflict** ก่อน
2. รองลงมา **Humor / Comedy Timing**
3. รองลงมา **Interaction Hook**
4. พิจารณาความหลากหลายของประเภท — ไม่ควรซ้ำประเภทกันเกิน 2 cut ติดกัน

### 4.2 จำนวน Cut ขั้นต่ำ-สูงสุด
| ความยาววิดีโอ | จำนวน Cut แนะนำ |
|---------------|----------------|
| < 30 นาที | 2-4 cut |
| 30-60 นาที | 3-6 cut |
| 1-2 ชม. | 4-8 cut |
| > 2 ชม. | 5-10 cut |

### 4.3 Minimum Viral Score
- Short-form: ต้องมี Viral Score ≥ 6.0/10
- Long-form: ต้องมี Viral Score ≥ 5.0/10
- ถ้าไม่มี segment ใดถึงเกณฑ์ ให้แจ้งผู้ใช้ว่า "ไม่พบช่วงที่เหมาะสม"

---

## 5. Quality Checklist (ก่อนส่งมอบ)

- [ ] ทุก cut มี **ประเภท (Category)** ที่เหมาะสม
- [ ] ทุก cut มี **ชื่อคลิป** ที่น่าสนใจและดึงดูดผู้ชม
- [ ] **Timestamp (HH:MM:SS)** ถูกต้องและอ้างอิงได้
- [ ] **ชื่อไฟล์นำเข้า (.mp4)** ถูกต้องตรงกับไฟล์ต้นฉบับ
- [ ] ไม่มี cut ใด **ซ้อนทับกัน** ใน timeline
- [ ] ความยาวคลิปเหมาะสมกับประเภท Short-form / Long-form
- [ ] ไม่มีเนื้อหาที่ละเมิดข้อกำหนดของแพลตฟอร์ม
- [ ] จำนวน cut รวมอยู่ในช่วงที่แนะนำ
- [ ] แต่ละ cut มีระยะห่างเพียงพอ ไม่กระจุกตัว

## 6. Output Format Compliance

ผลลัพธ์ต้องเป็นไปตาม Schema ใน [[Skills#5-output-schema]] เท่านั้น

## 7. AI Behavior Rules (ข้อปฏิบัติของ AI)

### 7.1 Git
- ห้ามใช้ Git commands (commit, push, pull, add, ฯลฯ) โดยเด็ดขาด หากผู้ใช้ยังไม่ได้บอกหรืออนุญาต
- หากจำเป็นต้องใช้ Git ให้แจ้งผู้ใช้ก่อนและรอคำยืนยัน

### 7.2 Auto-stop
- เมื่อแก้ไขไฟล์เสร็จ ให้หยุดรอคำสั่งจากผู้ใช้เท่านั้น อย่าดำเนินการใด ๆ ต่อโดยอัตโนมัติ

### 7.3 Post-task Updates (ทุกครั้งหลังทำงานเสร็จ)
**ทุกครั้งหลังทำงานเสร็จ ไม่ว่างานอะไรก็ตาม (Pipeline, Reference Clip, แก้ไขไฟล์, สร้างไฟล์ใหม่, ปรับปรุงเอกสาร) ต้องอัปเดตไฟล์ทั้งหมดต่อไปนี้:**

- [ ] **plan.md** — ติ๊ก `[x]` ทุก Step ที่ทำเสร็จแล้ว
- [ ] **References.md** — เพิ่ม reference ถ้ามี cuts / clips ใหม่
- [ ] **contents.md** — เพิ่มแถวเกมใหม่ ถ้ามี
- [ ] **chat-history.md** — **ทุกครั้ง** บันทึก session log การแชทของรอบนั้น
- [ ] **Personas.md** — อัปเดตข้อมูล Persona ถ้ามีข้อมูลใหม่
- [ ] **Welcome.md** — อัปเดตสถานะล่าสุดของโปรเจกต์

และตรวจสอบ consistency:

- [ ] `contents/` directory กับ `contents.md` — ตรงกัน
- [ ] ลิงค์ใน References.md — ชี้ไปไฟล์ที่มีอยู่จริง
